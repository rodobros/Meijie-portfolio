OnMo Design System

I was creating the app from scratch. In order for me to work efficiently and make sure the design would stay consistent, I created OnMo’s reusable and scalable design system from scratch. Below case study shows some elements of the design system. If you want to view them in details, please contact me.

Deliverables: Reusable and responsive component library, design documentation.

Tools: Figma

Duration: 2020

The Foundation

Building a design system is like building a house, we need to start with the foudations. And the foundations of a design system is the spacing and grid system. For the spacing and the grid system, I went with an 8pt grid system that consists of 4 column grid to ensure even numbers for scalability across screen sizes.

Typography/Fonts

After exploring different options, finally I decided to use Roboto as the main font because of the following reasons:

  1. This app is has a strong focus on imagery and graphics, the font cannot be too flashy.
  2. Roboto is easy-to-read and it doesn’t grab much attention.
  3. Roboto is a web font that is readily avaliable.
  4. Roboto also has the perfect weight options for this app, which makes the layout and visual herierchay looks balanced.

Iconography

For icons, I focused on simplicity instead of lots of details to avoid increasing cognitive load. The icons also have rounded corners to give a playful look since it’s a gaming platform.

Color

Colors should be selected carefully since they support the purpose of the content, communicating hierachy of inofrmation, interactive states, etc. OnMo’s color palette includes primary, secondary, color variants, background, states and texts.

Buttons

Since it is an application, there will only be inactive and loading states. For OnMo, I decided to have 4 different types of buttons: Primary, Secondary, Tertiary and Text Button. Below are some of the variations of the buttons.

Input Fields

Input fields are simiar to buttons, they need to have different states. When I was creating them, I focused on accessibility. All input field has a title, and there are more indicators than just colors for error states and success state, etc.

Illustrations

Similar to iconography, I created illustrations that are simple and easy to recognize.

Conclusion

Building a design system from scratch is very rewarding, I really enjoyed building this MVP design system because it’s also such an important aspect of design. The design system really helped me in the following ways:

  1. It made sure my design would stay consistant.
  2. It made me to take a step back and really think through the usage of each component, included when and where to use.
  3. It made the handoff process smooth.

What could I have done better?
Giving more time, I would love to build a complete design system which includes:

  1. More documentation about when and where to use components.
  2. Brand guidelines about the tone, voice, writing syle etc.
  3. Write the specs for components.
